4//! anvil-zksync
5//!
6//! The `anvil-zksync` crate provides an in-memory node designed primarily for local testing.
7//! It supports forking the state from other networks, making it a valuable tool for integration testing,
8//! bootloader and system contract testing, and prototyping.
9//!
10//! ## Overview
11//!
12//! - **In-Memory Database**: The node uses an in-memory database for storing state information,
13//!   and employs simplified hashmaps for tracking blocks and transactions.
14//!
15//! - **Forking**: In fork mode, the node fetches missing storage data from a remote source if not available locally.
16//!
17//! - **Remote Server Interaction**: The node can use the remote server (openchain) to resolve the ABI and topics
18//!   to human-readable names.
19//!
20//! - **Local Testing**: Designed for local testing, this node is not intended for production use.
21//!
22//! ## Features
23//!
24//! - Fork the state of mainnet, testnet, or a custom network.
25//! - Replay existing mainnet or testnet transactions.
26//! - Use local bootloader and system contracts.
27//! - Operate deterministically in non-fork mode.
28//! - Start quickly with pre-configured 'rich' accounts.
29//! - Resolve names of ABI functions and Events using openchain.
30//!
31//! ## Limitations
32//!
33//! - No communication between Layer 1 and Layer 2.
34//! - Many APIs are not yet implemented.
35//! - No support for accessing historical data.
36//! - Only one transaction allowed per Layer 1 batch.
37//!
38//! ## Usage
39//!
40//! To start the node, use the command `anvil-zksync run`. For more advanced functionalities like forking or
41//! replaying transactions, refer to the [official documentation](https://era.zksync.io/docs/tools/testing/anvil-zksync.html).
42//!
43//! ## Contributions
44//!
45//! Contributions to improve `anvil-zksync` are welcome. Please refer to the [contribution guidelines](https://github.com/matter-labs/anvil-zksync/blob/main/.github/CONTRIBUTING.md) for more details.
